# 主動學習 / 實驗設計（DoE）：先做哪些實驗最值得

> 把計算與實體實驗接起來：模型不能代替量測，但能回答「**手上資源有限，先合成哪幾個
> 樣品最能驗證/改進模型**」。這是 `MEASUREMENT_PROTOCOL.md` 的「選樣品」前置步驟。
> 程式：`alloy_engine/ga/active_learning.py`、`scripts/recommend_experiments.py`。

---

## 1. 實驗價值（acquisition）怎麼算

對候選成分池，每個候選的「值得做」分數 =

```
score = w_unc · (Tc、Br 的 MC-Dropout 不確定度，正規化後平均)
      + w_nov · (在特徵空間距訓練資料的最小距離，正規化)
```

- **不確定度**：模型越不確定 → 量測後學到越多（資訊量大）。用已校準 bundle 的
  MC-Dropout 標準差（Tc、Br）。
- **新穎度**：距 Tc/Br 真實訓練資料越遠 = 外推區 → 量測可擴展模型的可信域。
  （參考集用 MP Br 訓練組成；無外部資料時自動關閉，僅用不確定度。）

## 2. 兩個關鍵設計（避免常見陷阱）

1. **先過可製造性濾網**：純不確定度採樣會挑出模型外推最離譜但**無意義**的成分
   （如 La-Mo、P-Mn、負 Tc）。故先用 GA 化學懲罰（>0.5）濾掉不可製造/非磁性垃圾，
   只在**真實合金子空間**裡選——recommend 出來的才是可鑄、相關的樣品。
2. **多樣化批次**（greedy farthest-point）：不選 k 個幾乎相同的高分候選；在特徵空間
   讓選出的 k 個彼此最分散，避免做重複實驗。

## 3. 用法

```bash
python scripts/recommend_experiments.py --target-tc 150 --k 4
```

範例輸出（150°C 帶，已濾可製造性 + 新穎度啟用）：

| 預測Tc | ±K | 預測Br | 價值 | 理由｜主要成分 |
|---|---|---|---|---|
| 136°C | 71 | 0.74 | 1.03 | 成分新穎（外推）｜Gd-P-Fe |
| 180°C | 65 | 1.99 | 0.66 | 模型不確定度高｜Fe75-Ni24 |
| 143°C | 52 | 0.17 | 0.57 | 分散批次代表｜Fe-P-Ni-Mo |

→ 依此清單鑄樣 → VSM/DSC 量 Tc/Br → 比對預測。高不確定度 + 高新穎度的樣品，
量測後對模型修正最大。

## 4. 為何這是「計算 → 實驗」的橋

引擎已知自己**哪裡最不確定**（MC-Dropout）與**哪裡在外推**（離訓練資料）。主動學習
把這兩個自知之明，轉成「**用最少的實驗、最快驗證/改進模型**」的具體樣品清單——
讓你實驗室那一步（協定見 `MEASUREMENT_PROTOCOL.md`）花在刀口上，而非盲試。

## 5. 限制
- 不確定度為 MC-Dropout 代理，非校準後驗（量級可信、絕對值非嚴格機率）。
- 新穎度依賴是否提供訓練參考集（MP/NEMAD，git-ignored）。
- 仍是「建議先做什麼」，不是替代實驗。
